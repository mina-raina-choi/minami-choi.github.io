ElasticSearch 스냅샷 생성 및 복원

AWS elasticsearch에서 자동 백업을 지원하지만 이걸 복원하려면 AWS Support에 직접 요청하는 방식이라 원하는 시점에 복원하지 못할 수도 있다.
또한 elasticsearch 버전을 올리기위해 마이그레이션을 할 때는 새로운 elasticsearch를 생성하고 직접 백업 및 복원을 해야한다.
=> 수동으로 스냅샷 생성하고 복원하는 작업이 필요하다.

1. 준비사항

1-1. 스냅샷이 저장될 S3 만들기

  1. 버킷이름 정하기
  2. 되도록 해당 ES와 동일한 리전을 선택
  3. 나머지는 기본설정값으로

1-2. IAM Role 생성

  1. IAM -> Role(역할) - Create New Role(역할만들기)
  2. AWS 서비스 - EC2 - “다음”버튼
  3. Attach Policy에서는 아무것도 선택하지 않고 “다음”
  4. 검토 - 원하는 역할이름 (EsSnapshotRole)입력하고 - 역할만들기
  5. 신뢰관계 - 신뢰 관계 편집

1-3. IAM Policy 생성 후 연결

  1. 정책 생성 버튼
  2. 시각적편집기/JSON 중에서 JSON탭 선택
  3. 아래와 같은 데이터 넣음 (s3-bucket-name에 본인이 만든 버킷이름 넣음)
  1. 정책검토 클릭 - 정책 이름( EsCosmoshub2SnapshotPolicy) 입력 후, 정책 생성

1-4. IAM 정책과 역할 연결

  1. IAM - Role - 아까 만든 역할 클릭
  2. 정책연결 클릭
  3. 미리 만든 정책검색 - 체크박스 클릭 - 정책연결

2. 스냅샷 생성

2-1. 스냅샷 저장소 생성

AWS elasticsearch service에서는 스냅샷 저장소 생성을 curl 명령어로 할 수 없음.
python을 이용하거나 POSTMAN 이용.
포스트맨을 이용해보겠다.
포스트맨 Authorization설정에 type을 AWS Signature를 선택한 후 접근권한이 있는 계정 정보(AccessKey, SecretKey) 및 Advanced정보(AWS region, Service name)를 넣는다.

만약 위의 과정에서 아래와 같은 에러가 발생한다면 신뢰관계 - 신뢰 관계를 편집해줘야한다.

2-2. 스냅샷 생성


3. 스냅샷 복원

스냅샷 생성할 때와 마찬가지로 실행하자마자 반환값이 나오지만 뒤에서 계속 복원중인 상태.
복원 진행중일때는 클러스터 상태가 Red이고 원래의 클러스터 상태로 돌아오므로 클러스터 상태를 통해 복원이 완료됐는지 체크


[참고]
스냅샷 생성에 필요한 시간은 Amazon ES 도메인의 크기에 따라 다르다.
스냅샷 작업이 길게 실행되면 일반적으로 다음과 같은 오류가 발생한다.
“504 GATEWAY_TIMEOUT. “
elasticsearch에 저장하는 데이터가 보통 대용량이기 때문에 자주나타나는 에러다
무시하고 작업이 성공적으로 완료될 때까지 기다리면 된다.

3-1. 다른 elasticsearch로 스냅샷을 복원하는 경우

elasticsearch 를 마이그레이션 하거나 다른 elasticsearch 클러스터에 복원하려는 경우 새로운 elasticserach 에 스냅샷 저장소만 생성하면 된다. 스냅샷 저장소를 앞서 생성한 스냅샷 저장소와 동일한 옵션으로 생성하면 같은 S3를 바라보게 되므로 복원도 바로 된다
