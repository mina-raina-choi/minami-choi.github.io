코딜리티 MinMaxDivision 문제풀이

코딜리티에서 이분 탐색 문제 중 MinMaxDivision을 풀어보았다.

먼저 이분 탐색이 무엇인지 알아보자.

정렬된 배열 a가 주어졌을 때, 특정 요소 x가 배열 a에 존재하는지 체크하기 위해서 일일이 선형탐색(linear search)를 할 수도 있겠지만 이분 탐색(binary search)을 활용하면 좀 더 빠르게 검색할 수 있다.

이분 탐색이란?

정렬된 배열 a에 대해서 반복적으로 탐색 범위를 반으로 나눠가며 특정 요소를 찾는 방식이다. 보통 아래의 프로세스를 통해 구현하면 된다.

  1. 탐색하려는 배열의 최대값 high, 최소값 low, 중간값 mid를 잡아준다.
  2. 찾으려는 값 x와 중간값 mid의 값을 비교한다.
  3. 만약 x > mid 이면 중간값을 낮추어야 하므로 최대값을 high = mid - 1, 반대의 경우 low = mid + 1로 값을 할당한다.
  4. x == mid 이거나 high > low 일 때까지 2~3번을 반복한다.

위의 2~3번 과정이 반복될 때마다 탐색범위가 반으로 줄어들게 된다. 그러므로 시간복잡도는 O(logn)이 된다.

MinMaxDivision 문제를 풀어보자.

문제를 요약해보면 정수 N개로 이루어진 배열 A를 K조각으로 나눌 때의 부분 배열의 최대합 중 최소값을 구하라는 것이다.

N과 K는 정수 범위 [1..100,000]
M은 정수 범위 [0..10,000]
배열 A의 요소들은 정수 범위 [0..M]

아이고, 두야… 이 문제로 2시간 가까이 보낸 것 같다. 결국 100%까지는 구현 못하고 91%까지 통과했다. 일단 여기까지 진행된 내용을 기록에 남긴다.

이 문제를 풀 핵심 아이디어는 부분 배열의 합이 될 수 있는 최소값 low = 0, 최대값 high = 배열 A의 총합을 기준으로 중간값 mid를 구하고, mid를 부분 배열의 최대합으로 하는 부분 배열을 k 개 만들 수 있는지 체크한다.

A배열의 부분 배열을 K개 만큼 만들 수 있거나, k보다 적게 만들 수 있으면 mid를 작게 해봐야 함 -> high = mid를 줄여야한다.
반대의 경우 mid를 높여야하므로 low = mid + 1
이런 식으로 low < high이면 계속 반복한다.

통과하지 못한 케이스를 보면 제출한 답이 정답과 1씩 차이가 난다. 엣지케이스 처리를 잘못해 준 것 같다. 다시 살펴봐야겠다.
