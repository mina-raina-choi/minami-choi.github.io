모니터링 시스템 구축 - 프로메테우스

블록체인 노드가 문제 없이 잘 돌아가는지 모니터링을 하고 문제 상황 발생 시 알맞은 알림을 주기 위해 프로메테우스를 활용했다. 프로메테우스로 모니터링을 하기 위해 필요한 작업들을 기록으로 간단히 남기려고 한다.

사용한 툴들은 아래와 같다.

  • 모니터링 툴 : Prometheus
  • 시각화 툴 : Grafana (Prometheus가 간단한 그래프를 보여주지만, 실제 모니터링 환경에서는 좀 더 전문화된 시각화 도구가 필요)
  • Node Exporter : 주로 운영체제 커널에서 얻을 수 있는 machine-level metrics (CPU, memory, disk space, disk I/O, network bandwidth, 마더보드 온도)

위 그림은 내가 활용한 모니터링 시스템의 구조도이다.

대부분의 모니터링 도구가 Push 방식 즉, 각 서버에 클라이언트를 설치하고 이 클라이언트가 메트릭 데이터를 수집해서 서버로 보내면 서버가 모니터링 상태를 보여주는 방식을 취하는데, 프로메테우스는 Pull 방식을 사용하여, 서버에 클라이언트가 떠 있으면 서버가 주기적으로 클라이언트에 접속해서 데이터를 가져오는 방식으로 동작한다.

모니터링할 노드에 Node Exporter를 설치해서 프로메테우스 서버가 정보를 pull할 수 있도록 한다. 프로메테우스 서버는 정보를 수집하다가 alert.rules 에 정의한 규칙에 위반되는 상황이 발생하면 alertmanager를 통해 알람을 보낸다. 그리고 수집한 정보를 그라파나를 통해서 시각화를 해준다.

추가적으로 모니터링할 노드를 추가하려면 prometheus.yml 파일에 scrape_configs에 모니터링할 노드 정보를 넣으면 된다.

모니터링 알림 규칙을 추가하려면 alert_rules.yml 파일에 rules를 추가하면 된다.

[참고] 알람 규칙 예시 https://awesome-prometheus-alerts.grep.to/rules.html

/prometheus.yml파일에 정의된 노드에서 /alert_rules.yml에 정의된 알람 상황이 발생할 경우 어디에 어떻게 알람을 보낼지 /alertmanager.yml파일에 정리되어 있다.

아래는 최초로 모니터링 서버 구축시 필요한 과정들이다.

1. 프로메테우스 설치 (기본포트 9090)

1-1 설정파일 prometheus.yml

metrics를 수집할 서버 정보를 scrape_configs에 적어주면 된다.

[참고] 만약 일시적으로 특정 서버의 모니터링을 멈추고 싶다면 scrape_configs에서 주석처리한 후

sudo systemctl restart prometheus.service

1-2 alert_rules.yml

알람을 발생시킬 규칙 정의한다.

1-3 prometheus.service 등록

2. 그라파나 설치 (기본포트 3000)

3. 알람매니저 설치 (기본포트 9093)

3-1 alertmanager.yml

어떤 알람을 어떤 리시버에 보낼지 정의한다.

3-2. alert.service

alertmanager.yml 파일에 정의된대로 알람을 보낸다.

4. 노드 엑스포터 설치 (기본포트 9100)

모니터링할 노드별로 노드엑스포터를 설치해서 metrics를 수집하고 프로메테우스에 정보를 수집해 갈 수 있도록 한다.

4-1 node_exporter.service 등록
