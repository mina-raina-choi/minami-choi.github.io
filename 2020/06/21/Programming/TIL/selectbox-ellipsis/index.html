셀렉트 박스에서 text-overflow: ellipsis 구현하기

최근에 셀렉트박스에서 선택된 text가 overflow할 경우 ellisis 하도록 구현한 적이 있는데 잊기 전에 간단히 정리해보려고 한다.

일반적으로 text-overflow: ellipsis 속성은 block container element에 overflow 와 white-space 속성이 적용되어 있을 때 가능하다. 예를 들면,

그리고 text-overflow 속성은 block container element의 직렬 방향으로 넘치는 텍스트에 대해서만 영향을 미친다.
참고 https://developer.mozilla.org/en-US/docs/Web/CSS/text-overflow

하지만 위의 경우는 일반적인 경우고 select 박스에서 선택된 값에 대해 text-overflow: ellipsis를 적용시키기 위해서는 css만으로는 불가능하다. javascript의 도움이 필요하다.

메인 아이디어는 select 박스에서 선택된 값을 select 박스를 감싸는 container element의 data-content 어트리뷰트에 담아서 보여주는 것이다.

select 태그를 감싸는 SelectBox-wrapper에 선택된 값을 담을 data-content 어트리뷰트와 엘리먼트에 접근하기 위해서 ref에 selectBoxRef를 전달해준다.

select 태그에서 onChange 이벤트가 발생할 때, selectBoxRef.current.dataset.content에 value를 저장해주면 된다.

마지막으로 scss에서 해줘야 할 것은 SelectBox__select의 color를 투명하게 하고 SelectBox-wrapper의 pseudo element를 이용해서 data-content 어트리뷰트에 있는 값을 text-overflow: ellipsis 스타일을 입혀서 보여주는 것이다.

전체 코드 확인하기

하지만 끝날 때까지 끝난 것이 아니었다…

내 작업 환경인 맥os에서의 모던 웹 브라우저에서는 무사히 잘 작동했지만 윈도우 환경에서는 3가지 문제를 발견했다.

문제 1
윈도우 크롬에서 option들이 transparent 하게 나왔다.

이 문제는 간단하게 SelectBox__option에 color 값을 주므로써 해결했다.

문제 2
윈도우 ie11에서 select tag에서 color:transparent 가 먹지 않았던 것이었다!

Can I use 에서 확인해보면 IE9이상부터는 color:transparent를 지원하는 것으로 나오지만 ㅠㅠ 이상하게 select tag에서는 적용되지 않았다.

야매로 ie일 경우에는 흰 색을 적용하도록 했다.

문제 3
select tag에서 클릭 이벤트가 발생할 때마다 동적으로 data-content 값을 변경했는데, IE에서는 변경된 data-content 값이 화면에 그려지지 않고 처음 화면 렌더링이 될 때는 value가 계속 그대로 보이는 문제가 있었다.
-> 결국 IE에서는 select tag에서 text-overflow: ellipsis 하는 것을 포기했다!

끝!
